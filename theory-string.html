Theory: Strings

Definition

A string is a fundamental data type representing a sequence of characters. These characters can include letters, numbers, symbols, and whitespace. Strings are typically used to store and manipulate textual data in programming.

Key Characteristics

  • Sequence of Characters: Strings maintain the order of their characters. The first character is at index 0, the second at index 1, and so on.
  • Immutability (Commonly): In many popular languages (like Java, Python, JavaScript, C#), strings are immutable. This means that once a string object is created, its content cannot be changed. Operations that appear to modify a string (like concatenation or replacing characters) actually create and return a *new* string object. Some languages (like C/C++) handle strings as mutable arrays of characters.
  • Indexed Access: Individual characters within a string can often be accessed using their numerical index, similar to arrays (e.g., myString[0]).
  • Length Property: Strings usually have a property or method to determine their length (the number of characters they contain).
  • Character Encoding: Underlyingly, characters are represented by numerical codes (like ASCII or Unicode/UTF-8). This is usually handled transparently by the language but can be important for internationalization or complex text processing.

Time Complexity (Big O Notation)

  • Access Character (by index): Typically O(1) - Constant time.
  • Get Length: Typically O(1) - Constant time (length is often stored).
  • Concatenation (e.g., str1 + str2): O(n + m) - Linear time, where n and m are the lengths of the strings being combined. This is because a new string must be created, and characters from both original strings need to be copied into it (due to immutability). Repeated concatenation in a loop can be inefficient (O(n^2)). String builder/buffer classes can optimize this.
  • Search (for a character or simple substring): Typically O(n) or O(n*m) - Linear time (n = length of string, m = length of substring). More advanced algorithms exist but simple search often involves checking parts of the string.
  • Slicing/Substring Creation: Often O(k) - Linear time, where k is the length of the substring being created, as a new string object is usually formed.

Common Operations & Use Cases

  • Storing names, sentences, messages, code, file paths, URLs, configuration data.
  • Reading user input and displaying output.
  • Parsing text files (logs, CSV, JSON, XML).
  • String manipulation: searching, replacing, splitting, joining, changing case.
  • Data validation (e.g., checking email format).
  • Used as keys in dictionaries/hash maps.

Example (JavaScript)

// Creating strings
let greeting = "Hello";
let target = 'World';
let language = `JavaScript`; // Template literal

// Accessing characters (read-only due to immutability)
console.log(greeting[0]); // Output: H
console.log(language[4]); // Output: S

// Getting the length
console.log(target.length); // Output: 5

// Concatenation (creates a new string)
let message = greeting + ", " + target + "!"; // message is "Hello, World!"
console.log(message);
let originalGreeting = greeting; // Store original
greeting = greeting + " User"; // greeting is now a NEW string "Hello User"
console.log(greeting); // Output: Hello User
console.log(originalGreeting); // Output: Hello (original is unchanged)


// Common methods (most return NEW strings)
console.log(message.toUpperCase()); // Output: HELLO, WORLD!
console.log(message.toLowerCase()); // Output: hello, world!
console.log(message.includes("World")); // Output: true
console.log(message.indexOf("o")); // Output: 4 (index of the first 'o')
console.log(message.slice(0, 5)); // Output: Hello (extracts from index 0 up to, but not including, 5)
let words = message.split(" "); // Splits by space, returns ["Hello,", "World!"]
console.log(words);
