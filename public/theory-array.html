Theory: Arrays

Definition

Arrays are fundamental, linear data structures that store a collection of elements, typically of the same data type, in contiguous (adjacent) memory locations. Think of them like a numbered list of boxes, where each box holds one item and has a unique number (the index) starting from 0.

Key Characteristics

  • Fixed or Dynamic Size: Depending on the programming language, arrays might have a fixed size determined at creation (like in C/Java) or be dynamic, allowing them to grow or shrink (like Python lists or JavaScript arrays).
  • Indexed Access: Elements are accessed directly using their numerical index (e.g., `myArray[0]`, `myArray[5]`).
  • Contiguous Memory: Elements are stored one after another in memory, which allows for efficient iteration and cache performance.
  • Homogeneous (Usually): Most statically-typed languages require array elements to be of the same type (e.g., an array of integers, an array of strings). Dynamically-typed languages (like Python, JavaScript) often allow arrays to hold elements of different types.

Time Complexity (Big O Notation)

  • Access (by index): O(1) - Constant time. You can jump directly to any element if you know its index.
  • Search (by value): O(n) - Linear time. In the worst case, you might have to check every element to find a specific value.
  • Insertion (at end, if space available/dynamic): O(1) - Constant time (amortized for dynamic arrays).
  • Insertion (at beginning/middle): O(n) - Linear time. All subsequent elements need to be shifted to make space.
  • Deletion (at end): O(1) - Constant time.
  • Deletion (at beginning/middle): O(n) - Linear time. All subsequent elements need to be shifted to fill the gap.

Common Operations & Use Cases

  • Storing lists of items (e.g., user scores, product names).
  • Implementing other data structures (e.g., stacks, queues, hash table buckets).
  • Matrix operations (using 2D arrays).
  • Sorting and searching algorithms often operate on arrays.

Example (JavaScript)


// Creating an array
let numbers = [10, 20, 30, 40, 50];
let fruits = ["Apple", "Banana", "Cherry"];

// Accessing elements
console.log(numbers[0]); // Output: 10
console.log(fruits[2]); // Output: Cherry

// Modifying elements
numbers[1] = 25; // numbers is now [10, 25, 30, 40, 50]

// Getting the length
console.log(numbers.length); // Output: 5

// Adding elements (JavaScript arrays are dynamic)
fruits.push("Date"); // fruits is now ["Apple", "Banana", "Cherry", "Date"]

// Removing the last element
let lastFruit = fruits.pop(); // lastFruit is "Date", fruits is ["Apple", "Banana", "Cherry"]

// Iterating through an array
for (let i = 0; i < fruits.length; i++) {
    console.log(fruits[i]);
}
// Or using forEach
numbers.forEach(num => {
    console.log(num * 2);
});
