Theory: Sets

Definition

A Set is an abstract data type that represents a collection of **unique** elements, with no particular order. Think of it like a mathematical set – an element is either in the set or it isn't, and duplicates are automatically disregarded.

Key Characteristics

  • Uniqueness: A set cannot contain duplicate elements. If you try to add an element that already exists, the set remains unchanged.
  • Unordered (Usually): While some implementations might preserve insertion order (like JavaScript's `Set` or Python's `dict` keys since 3.7), the core concept of a set doesn't guarantee any specific order of elements during iteration.
  • Membership Testing: Sets are highly optimized for checking if a specific element is present in the collection.

Core Operations

  • Add: Adds an element to the set. If the element is already present, the set doesn't change.
  • Remove (or Delete): Removes a specific element from the set.
  • Contains (or Has): Checks if an element exists within the set. Returns true or false.
  • Size: Returns the number of unique elements in the set.
  • Set Operations (Common):
    • Union: Returns a new set containing all elements from two sets.
    • Intersection: Returns a new set containing only elements present in *both* sets.
    • Difference: Returns a new set containing elements from the first set that are *not* in the second set.
    • Subset/Superset Check: Determines if one set is contained within another.

Implementation

Sets are very often implemented using **Hash Tables** internally. This allows them to achieve fast average time complexity for core operations.

  • The elements themselves act as the keys in the underlying hash table.
  • The hash table structure enables efficient checking for existence (add, contains, remove).

Time Complexity (Typical Hash Table Implementation - Average Case)

  • Add: O(1)
  • Remove: O(1)
  • Contains: O(1)
  • Size: O(1) (if size is tracked), otherwise O(n)
  • Set Operations (Union, Intersection, Difference): Generally O(m + n) or O(min(m, n)) depending on the operation and implementation, where m and n are the sizes of the sets involved.

Time Complexity - Worst Case

(Due to hash collisions in the underlying hash table)

  • Add: O(n)
  • Remove: O(n)
  • Contains: O(n)

Common Use Cases

  • Removing Duplicates: Quickly get a collection of unique items from a list or array.
  • Membership Testing: Efficiently checking if an item has been seen before (e.g., in algorithms involving graphs or arrays).
  • Mathematical Set Operations: Performing union, intersection, difference etc.
  • Finding unique elements between collections.

Example (JavaScript - using Set)

// Creating a Set
let mySet = new Set();

// Adding elements
mySet.add(1);
mySet.add(5);
mySet.add("hello");
mySet.add(5); // Adding duplicate - ignored
console.log(mySet); // Output: Set(3) { 1, 5, 'hello' }

// Checking for existence
console.log(mySet.has(1)); // Output: true
console.log(mySet.has('world')); // Output: false

// Getting the size
console.log(mySet.size); // Output: 3

// Removing elements
mySet.delete(5);
console.log(mySet.has(5)); // Output: false
console.log(mySet.size); // Output: 2

// Creating from an array (removes duplicates)
let numbers = [2, 3, 4, 4, 2, 1, 3];
let uniqueNumbers = new Set(numbers);
console.log(uniqueNumbers); // Output: Set(4) { 2, 3, 4, 1 }

// Iterating (order generally matches insertion order in JS)
uniqueNumbers.forEach(num => {
  console.log(num * 10);
});
// Output (order might vary in other languages): 20, 30, 40, 10

for (let item of uniqueNumbers) {
  console.log(item);
}
// Output: 2, 3, 4, 1
