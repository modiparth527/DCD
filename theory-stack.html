Theory: Stacks

Definition

A Stack is a linear data structure that follows the **LIFO (Last-In, First-Out)** principle. Imagine a stack of plates: you can only add a new plate to the top, and you can only remove the topmost plate. The last plate placed on the stack is the first one to be removed.

Core Operations

  • Push: Adds an element to the top of the stack.
  • Pop: Removes and returns the element from the top of the stack. An error or specific value might be returned if the stack is empty.
  • Peek (or Top): Returns the element at the top of the stack without removing it. An error or specific value might be returned if the stack is empty.
  • isEmpty: Checks if the stack contains any elements. Returns true if empty, false otherwise.
  • Size: Returns the number of elements currently in the stack.

Implementation

Stacks can be easily implemented using underlying data structures:

  • Arrays/Dynamic Arrays (Lists): Using a dynamic array (like JavaScript arrays or Python lists) is common. push corresponds to appending to the end, pop corresponds to removing from the end. These operations are typically amortized O(1).
  • Linked Lists: A linked list can also implement a stack. push involves adding a new node at the head, and pop involves removing the head node. These are typically O(1).

Time Complexity (Typical Implementations)

  • Push: O(1) (Amortized O(1) for dynamic arrays)
  • Pop: O(1)
  • Peek: O(1)
  • isEmpty: O(1)
  • Size: O(1)
  • Search (by value): O(n) - Stacks are not designed for efficient searching.

Common Use Cases

  • Function Call Stack: Programming languages use a stack to manage function calls (storing local variables, return addresses).
  • Undo/Redo Features: Storing previous states to allow undoing actions.
  • Expression Evaluation: Converting infix expressions to postfix (Reverse Polish Notation) and evaluating postfix expressions.
  • Parsing: Used in algorithms for parsing syntax (like checking for balanced parentheses).
  • Backtracking Algorithms: Managing states in algorithms that explore possibilities (e.g., depth-first search, maze solving).
  • Browser History (Forward/Back): Conceptually similar, though often more complex in practice.

Example (JavaScript - using Array)

let stack = []; // Use an array as a stack

// Push elements
stack.push('A');
stack.push('B');
stack.push('C');
console.log(stack); // Output: ['A', 'B', 'C']

// Peek (access the last element)
console.log(stack[stack.length - 1]); // Output: C

// Pop elements
let topElement = stack.pop();
console.log(topElement); // Output: C
console.log(stack); // Output: ['A', 'B']

topElement = stack.pop();
console.log(topElement); // Output: B
console.log(stack); // Output: ['A']

// Check if empty
console.log(stack.length === 0); // Output: false

stack.pop(); // Remove 'A'
console.log(stack.length === 0); // Output: true

// Trying to pop an empty stack
let emptyPop = stack.pop();
console.log(emptyPop); // Output: undefined
